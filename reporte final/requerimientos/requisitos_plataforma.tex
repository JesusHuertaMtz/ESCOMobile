\subsection{Requisitos de plataforma}
Los requerimientos de plataforma son la base sobre la cual se construirá el sistema y definen
cuales son las tecnologías a utilizar para desarrollar el sistema.

\begin{requisitos}{RFN1 Plataforma móvil.}
	\RFitem{Descripción}{La aplicación se ejecutará sobre la plataforma móvil Android para smartphones.
	En la versión Android 5.0 (Lollipop) a la versión Android 8.0 (Oreo).}
	\RFitem{Propósito}{Para que una gran parte, sino es que, en su totalidad, de la comunidad de
	ESCOM disponga de ESCOMobile en sus dispositivos con S.O. Android.}
	\RFitem{Usuario}{N/A.}
	\RFitem{Tipo}{Móvil.}
\end{requisitos}

\begin{requisitos}{RFN2 Lenguaje de programación.}
	\RFitem{Descripción}{La aplicación ESCOMobile se escribirá en el lenguaje de programación 
	Kotlin versión 1.2.X}
	\RFitem{Propósito}{
		Estar a la vanguardia tecnológica ya que Kotlin es el lenguaje oficial para programar aplicaciónes 
		para Android.
		A continuación se muestran algunas de las ventajas que ofrece Kotlin ante el lenguaje de 
		programación Java.
		\begin{itemize}
			\item Es totalmente compatible con Java.
			\item Se actualiza periodicamente.
			\item Escribir menos código para realizar las mismas funciones que Java.
			\item Se puede mezclar con código de Java sin afectar la funcionalidad.
			\item Es seguro frente a NullPointerException.
		\end{itemize} 
	}
	\RFitem{Usuario}{N/A.}
	\RFitem{Tipo}{Móvil.}
\end{requisitos}
